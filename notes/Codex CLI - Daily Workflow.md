# Codex CLI - Daily Workflow

Back to [[Codex CLI Fundamentals]].

A solid workflow:

1. Start in the right directory.
2. State the goal and constraints.
3. Let Codex inspect the project.
4. Ask for a plan if the change is risky.
5. Let Codex edit files.
6. Let Codex run focused checks.
7. Review the diff yourself.
8. Ask for refinements.
9. Commit when the change is coherent.

Example:

```text
Find why the invoice PDF test is failing. Do not change public API behavior. Prefer a minimal fix. Run only the relevant test first.
```
